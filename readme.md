# BMap Audio Input Assistant

**BMap Audio Input Assistant** 是一个 Windows 桌面自动化辅助工具。

它通过 ADB (Android Debug Bridge) 模拟人工点击和虚拟音频输入，辅助用户完成百度地图语音包录制过程中繁琐的重复性操作。本工具旨在用于**音频通道测试**及**自动化流程研究**。

## ✨ 特性 (v1.3.1)

*   **自动化录入**：自动识别屏幕文本，调用 TTS 播放音频并点击录音按钮，解放双手。
*   **IndexTTS 集成**：支持对接本地 TTS 服务，实现“文本-语音”直通车。
*   **非侵入式**：不修改 APP 任何文件，不读取 APP 隐私数据，完全模拟物理操作。

## 🛠️ 环境依赖

运行前请确保满足以下硬性条件：

1.  **音频驱动 (必装)**：安装 [VB-Audio Virtual Cable](https://vb-audio.com/Cable/) 并重启。
    *   安装后需配置**侦听功能**（否则录制时听不到TTS音频）：
        1.  右键任务栏音量图标 -> **声音** -> **录制**选项卡
        2.  右键 `CABLE Output` -> **属性** -> **侦听**选项卡
        3.  勾选 **侦听此设备**，下方选择你的播放设备（如耳机/USB声卡/扬声器）
        4.  点击**确定**保存
2.  **IndexTTS 服务**：本地部署并可运行。
3.  **安卓模拟器 (雷电/MuMu)**：
    *   **ROOT 权限**：建议开启 (便于录制结束后，您手动在系统目录中查找文件)。
    *   **分辨率**：建议 `720x1280` (DPI 320)。
    *   **App**：安装百度地图并登录。

> [!IMPORTANT]
> **⚠️ 核心注意事项 (必读)**
>
> 模拟器的音频输入通道会在重启后重置。**每次启动模拟器后，必须手动执行以下操作：**
> 1. 进入模拟器设置 -> **音频设置**。
> 2. 将 **麦克风输入 (Mic)** 修改为 `CABLE Output (VB-Audio Virtual Cable)`。
> 3. **保存设置**。
>
> **后果：如果不设置，软件播放的声音无法传入模拟器，导致录制全是静音。**

> [!NOTE]
> **关于 Root 权限**
> 本工具的自动化录入功能**不需要** Root 权限。
> 但如果您需要在录制完成后手动管理或导出产生的文件，通常需要模拟器开启 Root 权限以访问系统目录。

## 🚀 使用指南

1.  **启动服务**：配置 IndexTTS 和音频设备。
2.  **准备模拟器**：将麦克风指向 `CABLE Output`，并手动进入录音界面。
3.  **自动运行**：点击 **▶ 开始自动录制**，软件将辅助您完成后续的点击和朗读工作。
4.  **结束**：录制完成后，请用户自行在模拟器中进行后续操作（如保存、试听等）。

## ❓ 常见问题

**Q: 录完的文件在哪里？**
A: 本工具不提供文件提取功能。请在安卓系统的文件管理器中自行查找（通常在 `Android/data/com.baidu.BaiduMap/...` 目录下），具体路径请参考安卓系统规范。

**Q: 为什么生成的语音包无法导出？**
A: 本工具仅负责“录入”。导出功能受限于 APP 自身的逻辑和安卓权限，请利用模拟器自带功能手动操作。
